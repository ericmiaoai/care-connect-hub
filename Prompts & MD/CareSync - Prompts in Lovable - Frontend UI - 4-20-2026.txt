I am building a React 19 web application called "CareSync". Please read the attached Product Requirements Document (CareSync_PRD_MVP.md) for the full architectural context. 

Your task is to generate the initial frontend UI shell, the global design system, and the core module views. Do not build a backend; mock the data using local state so I can visualize the interactions before hooking it up to Supabase.

### 1. Tech Stack & Design System
* **Framework:** React, Vite, Tailwind CSS, Framer Motion, Lucide React (for icons).
* **Design Language:** Apple Human Interface Guidelines (HIG). 
* **Theme:** Strict Dark Mode. Use deep, minimal grays (`bg-zinc-950` or `bg-black`) with elevated cards (`bg-zinc-900`). Borders should be subtle (`border-zinc-800`).
* **Typography:** Clean, sans-serif (Inter or system-ui), high-contrast white/off-white text.
* **Color-Coding System (Cozi-style):** Implement a global color palette for users. Assign distinct Tailwind colors to mock users (e.g., Mom = blue-500, Nurse = emerald-500). Apply these colors as thin left-borders on cards or small avatar dots to indicate who is assigned to a task without relying purely on text.
* **Accessibility:** All buttons and interactive elements must be large touch targets (minimum 44x44px) to accommodate exhausted or older caregivers.

### 2. Global Layout & Mandatory Elements
* **Navigation:** A simple bottom tab navigation bar for mobile, adapting to a left sidebar on desktop. Tabs: "My Day", "Calendar", "Updates", "Scan AVS".
* **Regulatory Disclaimer:** At the very bottom of the app shell, include a persistent, muted, small-text disclaimer: "CareSync is an administrative organizational tool, not a substitute for professional medical advice."

### 3. Core Views to Generate

**View A: The Clinical "My Day" Dashboard (Any.do Style)**
* A 24-hour focused view. Display a vertical timeline of mock medical tasks (e.g., "08:00 AM - Lisinopril 10mg").
* Each task should be a clean card with a large checkbox.
* **Interaction:** When a user checks a box, animate its removal using Framer Motion and immediately trigger a bottom toast notification that says "Task Completed" with a prominent "Undo" button (lasting 5 seconds).

**View B: Shared Care Calendar**
* A clean, monthly or weekly grid. 
* Populate it with color-coded mock events (e.g., a green pill icon for medication, a blue car icon for transport). Do not make this cluttered.

**View C: Status Update Board**
* A simple, read-only feed where the Admin posts updates. 
* Cards should look like clean text messages or micro-blog posts with a timestamp and the Admin's name.

**View D: The AI Staging UI (Human-In-The-Loop)**
* This is the most critical safety view. 
* Top: A large, dashed-border "Upload After Visit Summary (AVS)" dropzone area.
* Middle: A "Parsed Results" table. Mock this by showing structured JSON data rendered into a clean UI table (Medication, Dosage, Time).
* Bottom: A massive, primary "Approve & Add to Schedule" button, and a secondary "Reject/Edit" button.

Please generate the foundational layout, routing, and these four views using mock data to demonstrate the color-coding and Framer Motion "Undo" interactions.
