Layer 1
User & UI
What the user sees and does
👤Sign up

User opens /register and fills in name, email, password.

✉️Verify email

User clicks confirmation link sent by Supabase Auth (gotrue).

⭕Create Care Circle

In onboarding, user names the circle and adds the patient.

🔗Join via invite

User opens /join?code=… and enters the 4-digit PIN.

➕Add task

My Day → Add Task. Fills title, time, priority, assignee.

📅Add appointment

Calendar → Add Appointment. Fills date, time, location, who's going.

📢Post broadcast

Admin composes update + severity (Info/Warning/Critical).

📷Take AVS photo

Scan AVS → camera. User photographs an After-Visit Summary.

✏️Review extracted

HITL screen — user inspects each field, edits if needed, taps Approve.

⚠️Delete account

Settings → Danger Zone → Delete Account → confirmation dialog.

Layer 4
Supabase
Auth · Postgres · Realtime · Storage
🔐auth.users INSERT

Supabase Auth (gotrue) hashes the password with bcrypt and creates the user row.

⚡profiles trigger

DB trigger auto-creates a profile row referencing the new user id.

handle_new_user()
📧Confirmation email

Auth dispatches the email with a one-time confirmation link (24-hour validity).

🎟️JWT session

After confirmation, Auth issues an access token (1h) + refresh token to the browser.

⭕care_circles INSERT

Row created with name + created_by. RLS guards reads to circle members only.

👥care_circle_members INSERT

Adds the user → circle relationship with their role (admin/collaborator/viewer).

🎫Invite validate

RPC checks invite code + PIN match, not expired, single-use → grants membership.

📋tasks INSERT

Row inserted with title, due_date, priority, assigned_to, created_by, care_circle_id.

📅calendar_events INSERT

Appointment row created with start/end times, location, patient_id, care_circle_id.

📢broadcast_updates INSERT

Author, content, severity. RLS limits inserts to Admins of the circle.

📜avs_documents audit

Non-PHI audit row (visit date, provider, reviewer, timestamps). No medications or clinical text.

🔄Realtime broadcast

Postgres publication emits change event → WebSocket pushes to every subscribed member's app.

supabase_realtime
🛡️Guardrail query

Server queries circles where user is sole admin with other members → returns blocking list.

🧹Orphan circles DELETE

Sole-member circles deleted → CASCADE wipes their tasks, events, broadcasts, AVS records.

💥auth.users DELETE

Admin API removes the user row → CASCADE deletes profile, memberships. Tasks/events created by them stay (SET NULL).
